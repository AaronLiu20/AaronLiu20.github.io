Current Research and Resume

Validation-suite

Being currently employed through the university of Delaware in a research position validating OpenACC code. OpenACC is a coding language that ineracts with both the CPU and the GPU by serving as the bridge between the two parts of the computer. This is because when use OpenACC to declare a region of code, it would compile the section of code onto the GPU. If this requires data to be moved from the host, CPU, into the GPU, then you declare it initially when you declare the region. It is able to accelerate the code since it could run different sections at the same time, parralelly. Being able to intregrate OpenAcc into any pre-exiting code, it is widely used in real-world applications to accelerate its performance.

As a reasearcher, I’m testing the functionality of OpenAcc by reading the documentation and implementing it. If the implementation fails in any way, document the errors and return it to Nvidia. Projected to stay on the team, I will be reasearch throughout the senior year and the winter/summer breaks
